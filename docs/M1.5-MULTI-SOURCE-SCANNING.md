# M1.5 — Multi-Source Scanning + Dictionary Growth

**Status:** Planning
**Depends on:** M1 (complete)
**Exit criteria:** OBF grooming barcodes resolve and score. Ingredient dictionary covers top grooming + supplement compounds (~300 entries). Supplements resolve from DB when pre-populated.

---

## Goal

Extend the scan endpoint beyond OFF-only to support grooming products via Open Beauty Facts (OBF) and lay the groundwork for supplements via DSLD. Expand the ingredient dictionary from ~60 to ~300 curated entries so grooming scans produce meaningful scores.

---

## What Changes

### 1. OBF Source Adapter

**New file: `lib/sources/openbeautyfacts.ts`**

Mirrors `openfoodfacts.ts` exactly — Zod schema, typed error class, async fetcher. OBF's API is 98% identical to OFF (same maintainers, same data format). Key differences:
- Base URL: `https://world.openbeautyfacts.org/api/v2/product/{barcode}.json`
- No `nutriscore_score` or `nutrition_grades` fields (beauty products don't have Nutri-Score)
- Reuses `OFF_USER_AGENT` env var
- 8s timeout, returns null on 404

### 2. DSLD Source Adapter

**New file: `lib/sources/dsld.ts`**

DSLD (NIH Dietary Supplement Label Database) has **no barcode lookup endpoint** — you can only search by name/brand or fetch by internal ID. This means:
- At scan time, supplements are only found if already in our DB (populated by M2 weekly sync or manual seed)
- This adapter exposes `searchDsld()` and `fetchDsldLabel()` for use by the M2 cron job
- Also includes `normalizeDsldUpc()` to convert DSLD's spaced UPC format ("0 48107 05843 2" → "048107058432")

**New file: `lib/utils/retry.ts`** — simple exponential backoff wrapper (3 attempts). DSLD returns intermittent 500s per the SOP.

### 3. Normalization Layer Update

**Modified file: `lib/normalize.ts`**

Add two new normalizers alongside existing `normalizeOffProduct()`:
- `normalizeObfProduct(obf, barcode)` — category always `'grooming'`, no nutriscore, id prefix `obf:`
- `normalizeDsldLabel(label, barcode)` — category always `'supplement'`, parses `ingredientRows` + `otheringredients`, ingredient_source `'verified'`, id prefix `dsld:`

Both reuse the existing shared helpers: `normalizeIngredientName()`, `lookupIngredientFlag()`, `determineCompleteness()`.

### 4. Scan Route — Multi-Source Cascade

**Modified file: `app/api/products/scan/[barcode]/route.ts`**

New lookup flow:
```
1. DB cache check (any source) → serve if found with score
2. OFF + OBF in parallel (Promise.allSettled) → first non-null wins
   - OFF hit → normalize as food, score, return
   - OBF hit → normalize as grooming, score, return
3. 404 with { capture: true }
```

Parallel OFF+OBF saves ~1s on cold misses. Supplements only resolve from DB cache (step 1) until M2's weekly sync populates them.

The `after()` callback also starts writing `product_ingredients` rows (not just the product row) so future cache hits can reconstruct the full product.

### 5. Dictionary Expansion

**Modified file: `lib/dictionary/seed.ts`**

Expand from ~60 to ~300 entries. Organized by section:

| Section | Count | Examples |
|---|---|---|
| Grooming — negative | ~40 | Formaldehyde releasers, PEGs, phthalates, coal tar dyes |
| Grooming — caution | ~30 | Silicones, additional sulfates, aluminum compounds |
| Grooming — positive | ~25 | Ceramides, peptides, squalane, panthenol |
| Supplement — positive | ~40 | Vitamins A-K, zinc, magnesium, ashwagandha, omega-3 |
| Supplement — caution | ~15 | Proprietary blends, mega-dose vitamins, TiO2 (oral) |
| Food — expanded | ~40 | Seed oils, emulsifiers, additional preservatives |
| Cross-category | ~20 | Water, citric acid, ascorbic acid, glycerin |

Every entry carries an `evidence_url` (PubMed, CIR, SCCS, NIH, FDA). Unknown ingredients stay neutral per charter §13.2.

**New file: `scripts/seed-dictionary.ts`** — one-off script to upsert seed entries into the `ingredient_dictionary` DB table: `npx tsx scripts/seed-dictionary.ts`

### 6. Type Update

**Modified file: `types/guardscan.ts`**

Add `subcategory: string | null` to `Product`. Non-breaking (nullable, defaults to null). Populated by M2.5's subcategory inference.
